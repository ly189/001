# Compile-Engine v3.1 Test Suite

> 用途：验证 v3.1 各增量在真实任务上的触发率与误触发率
> 配套：`Compile-Engine-v3.1.md`
> 决策依据：单 TC 总分 ≥ 6/8 → 保留；单增量误触发 ≥ 3 次 → 收紧或降级

---

## 评分指标（每用例 0–2 分，满分 8/case）

| 维度 | 0 | 1 | 2 |
|---|---|---|---|
| **正确触发** | 应触发的增量未触发 | 部分触发 | 完整触发 |
| **避免误触发** | 触发了不该触发的增量 | 弱误触发 | 完全避免 |
| **Token 经济性** | 默认路径明显变重 | 略增 | 无增加 |
| **输出可用性** | 不可直接用 | 需修改 | 可直接复制使用 |

---

## 测试矩阵（12 用例）

### TC-01｜Simple 直出（反例 — 测默认路径零开销）
- **输入**：写个 bash 命令删除当前目录下所有 .DS_Store 文件
- **应触发**：无（直接结果）
- **禁触发**：澄清、结构化交付、风格说明、Inputs Locked
- **目的**：验证默认路径不被任何 v3.1 增量污染

### TC-02｜Medium 单点澄清
- **输入**：帮我写一封求职信
- **应触发**：1–2 问（目标公司？岗位？）+ §8 风格自适应
- **禁触发**：Strategic 二选一、多假设三角化、Inputs Locked

### TC-03｜Strategic 全套
- **输入**：我想做副业，怎么开始
- **应触发**：
  - §3 微型确认核心目标
  - §3 优化变量识别 → §2 Strategic +1 二选一
  - §8 冷启动微提示
- **禁触发**：直接给方案、超过 3 问

### TC-04｜输入悖论（新能力 — §4 反向约束提问）
- **输入**：给我写一篇极度详细的产品文档，字数控制在 200 字内
- **应触发**：暴露"详尽 vs 字数"悖论，让用户取舍
- **禁触发**：自行妥协后直接出稿

### TC-05｜多假设三角化（正例）
- **输入**：帮我学 Python
- **应触发**：识别 ≥2 种解读（语法入门 / 解决具体问题 / 选资源）+ 三角化提问
- **禁触发**：直接给学习路线（属于"把推测当事实"）

### TC-06｜Divergent 模式（新能力 — §2 模式维度）
- **输入**：帮我给一个 AI 笔记 App 起名，给我多个方向
- **应触发**：Divergent 模式 + 多个并列方案 + [推测] 显式标记
- **禁触发**：收敛到单一最优名字

### TC-07｜主动索取 Few-Shot（新能力 — §4）
- **输入**：帮我写一首关于秋天的诗，要那种我朋友圈风格
- **应触发**：主动索取 1 个微型样例（"能给一句你心目中的好样例吗？"）
- **禁触发**：直接生成 / 自行脑补"朋友圈风格"

### TC-08｜不可逆高风险 +1（新能力 — §4）
- **输入**：我准备辞职 all in 创业，给我个建议
- **应触发**：澄清 +1 + 显式说明"决策不可逆，需多确认一项"
- **禁触发**：直接给建议、流程表演

### TC-09｜多轮接续（新能力 — §8）
- 轮1：我想做一个英语学习 App
- 轮2：针对在职白领，每天 15 分钟
- **应触发**：轮 2 显示 `[已锁定] Goal`，`[本轮新增]` 受众/时长，不重复问目标
- **禁触发**：轮 2 重新询问"你的目标是什么"

### TC-10｜抗流程表演反例（测不过度）
- **输入**：详细分析这段代码做什么 `return [x for x in seq if x > 0]`
- **应触发**：直接一两句解释
- **禁触发**：Inputs Locked、Key Decisions、CoT 表演、降级模板

### TC-11｜多假设三角化误触发反例
- **输入**：给我推荐一个数据库
- **应触发**：1–2 问（用例？规模？）
- **禁触发**：三角化（虽然候选多，但"推荐"已锁定任务方向，无 ≥30% 后果偏离）

### TC-12｜极度模糊 → 降级输出（新能力 — §7）
- **输入**：帮我做个项目，加油！
- **应触发**：降级输出模板（Draft + Open Risks + Verification Checklist）
- **禁触发**：硬要澄清到第 5 问、强行编造 Goal

## 决策规则

1. **保留**：单 TC 总分 ≥ 6/8，且增量正常触发
2. **收紧触发条件**：误触发 ≥ 3 次（在不该触发的 TC 上触发）
3. **增强触发信号**：应触发但未触发 ≥ 3 次
4. **降级为可选**：保留但默认关闭，需用户显式开启
5. **淘汰**：误触发率 > 50% 且无法收紧

---

## 使用方法

### 方式 A：模拟跑（快速）
让一个 LLM 加载 `Compile-Engine-v3.1.md` 作为 system prompt，
依次输入 12 个 TC，记录响应，按指标评分。

### 方式 B：真实跑（高保真）
将 v3.1 部署到实际工作流（如 Cursor / Claude Projects / Custom GPT），
用真实任务对照 TC 表评分。

### 方式 C：A/B 对比（最严格）
同时跑 v3 和 v3.1，比较同一 TC 下两版本的输出，
确认 v3.1 增量在该 TC 上是否真正提升（而非仅"看起来更复杂"）。
