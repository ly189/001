# Compile-Engine v3.1.1 Test Suite

> 用途：验证 v3.1.1 探索型架构的触发率与不退化保证
> 配套：`Compile-Engine-v3.1.1.md`
> 基础测试：`Compile-Engine-v3.1-test-suite.md` 的 12 个 TC 仍适用，需修改 TC-03 预期
> 决策依据：单 TC 总分 ≥ 6/8 → 保留；不退化 TC（TC-01/08/10）必须维持 v3.1 原分数

## v3.1 → v3.1.1 测试用例变更

### 必修改：TC-03（预期完全不同）

**输入**（不变）：我想做副业，怎么开始

**v3.1 旧预期**：直接二选一硬收敛（A 现金流 / B 长期资产）

**v3.1.1 新预期**：
- 阶段识别 → exploring（"想做"无决策语言）
- §4.5 Possibility Space 触发，暴露 4–5 个副业方向
- 输出 Working Hypothesis（可调整）
- 末尾保留"还有别的方向？或回复'出方案'"开放出口
- **禁止**强制选 A/B 主优化变量

### 不变：TC-01 / TC-02 / TC-04 / TC-05 / TC-06 / TC-07 / TC-09 / TC-10 / TC-11 / TC-12
全部保持 v3.1 测试套件的预期不变。

### 关键不退化验证：TC-08

**输入**（不变）：我准备辞职 all in 创业，给我个建议

**v3.1.1 新预期**：
- 阶段识别 → executing（"我准备" + "辞职" = 决策语言）
- 跳过 exploring，直接进 executing
- 触发不可逆 +1（M-4 不抑制，因已不在 exploring）
- 行为与 v3.1 一致，**不能因为加了探索保护而退化**

## 新增 TC（v3.1.1 专项）

### TC-13｜探索语言识别（exploring 触发）

- **输入**：我在想要不要学一下 AI，能给点建议吗？
- **应触发**：
  - 阶段识别 → exploring（"在想" + "要不要" = 强探索语言）
  - §4.5 Possibility Space：暴露 AI 学习的 3–5 个不同方向
    （例：用 AI / 学 AI 应用 / 学 AI 技术 / 用 AI 创业 / 用 AI 研究）
  - Working Hypothesis 软锁定
  - 开放出口（"还有别的方向？"）
- **禁触发**：
  - Inputs Locked（exploring 阶段不应强锁）
  - 主优化变量强制二选一（M-5 抑制）
  - 不可逆 +1（M-4 抑制）
  - 直接给 AI 学习路线（属于过早收敛）

### TC-14｜Possibility Space 完整触发

- **输入**：我想做内容创作
- **应触发**：
  - 阶段识别 → exploring
  - §4.5 Possibility Space 标准格式：
    - 4–5 个方向（图文/短视频/长视频/播客/电子书 等）
    - 每个方向配 1 句"关键问题是什么"
    - 末尾开放出口
  - **禁止**自行加权排序方向
- **禁触发**：
  - 直接推荐"短视频是趋势"（自行排序）
  - 列 8+ 方向（信息过载）
  - 每方向只给名字不给"关键问题"

### TC-15｜exploring → executing 切换（关键逃生入口）

- **轮 1 输入**：我想了解一下做独立开发者的可能性
- **轮 1 应触发**：
  - exploring 阶段
  - Possibility Space 暴露独立开发的几个方向
- **轮 2 输入**：直接出方案，按 SaaS 方向给我
- **轮 2 应触发**：
  - 阶段切换 → executing（"直接出方案" = 显式执行指令）
  - 跳过 converging，立即进 executing
  - 用 Inputs Locked 三段式（不再用 Working Hypothesis）
  - 多轮接续协议：轮 1 的 Working Hypothesis 应升级为轮 2 的 [已锁定]
- **禁触发**：
  - 继续在 exploring（用户已显式要求执行）
  - 重新询问已经在轮 1 暴露过的维度

### TC-16｜决策语言强切换（不退化保证 #2）

- **输入**：我已经决定要做一个 AI 笔记 App，给我 30 天启动方案
- **应触发**：
  - 阶段识别 → executing（"已经决定" + "给我...方案" = 双重决策/执行信号）
  - **跳过 exploring**，不触发 Possibility Space
  - 直接 Inputs Locked + Optimized Prompt
  - 行为与 v3.1 完全一致
- **禁触发**：
  - Possibility Space（用户已锁定方向）
  - "你考虑过其他类型 App 吗？"（违反用户已决策的事实）

### TC-17｜抗探索保护过度化（不退化保证 #3）

- **输入**：写一封英文 cold email 给 YC partner，介绍我做的 AI 工具
- **应触发**：
  - 阶段识别 → executing（明确执行任务，无探索语言）
  - 复杂度 = Medium
  - 1–2 问（你的工具是 / partner 是谁等）
  - 直接生成 cold email
- **禁触发**：
  - Possibility Space（"你考虑过其他写作风格吗？"）
  - Working Hypothesis（任务明确，无需软锁）
  - 任何"探索保护"过度化

## 使用方法

### 方式 A：模拟跑（快速）
让一个 LLM 加载 `Compile-Engine-v3.1.1.md` 作为 system prompt，
依次输入 17 个 TC，记录响应，按指标评分。

### 方式 B：A/B 对比（强烈推荐）
**同一组 TC** 分别跑 v3.1 和 v3.1.1，
重点比较：
- TC-03（应有显著差异，v3.1.1 在 exploring 场景显著更好）
- TC-08 / TC-16 / TC-17（不应有差异，验证不退化）

### 方式 C：真实跑
将 v3.1.1 部署到实际工作流，用真实任务对照 TC 表评分。
特别记录"用户是否感到被过度引导"或"用户是否感到啰嗦"。

## 决策规则

1. **保留**：单 TC 总分 ≥ 6/8，且增量正常触发
2. **不退化保证失守** → v3.1.1 退回 v3.1，重新设计阶段识别
3. **探索保护误触发率 > 20%** → 收紧 §3.5 阶段识别信号
4. **Possibility Space 信息过载（用户反馈）** → 收紧到 3 个方向上限
